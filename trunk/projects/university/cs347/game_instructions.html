CS347 SP2010 Game Project Page - Backgammon

Revision 2/23/2010. Note: it is the students' responsibility to check the CS347 website for possible new revisions!

The game this semester is Backgammon. To complete the game project assignments, you need to fill in the run() function of the AI class in the provided AI framework template to implement an AI player for Backgammon. Upon each call of the run() function, your AI player needs to return the best legal move it can find using the search algorithm specified in the below phase grid. The latest version of the framework can be checked out from Google Code using the following command:

svn checkout http://cs347backgammon.googlecode.com/svn/trunk/ cs347backgammon-read-only

Further instructions and documentation regarding the framework will be found in the root directory of this svn repository.

You should only submit a single zip or tar.gz file (not bzip or any other compression format!) containing all of the files required to compile/execute your program, on or before the due date specified in the below phase grid. This is accomplished by following the instructions at http://web.mst.edu/~tauritzd/submission.html.

Note: this is an individual project; plagiarism of any type will result in a zero grade.

While it is hoped that all code will be submitted error-free and on time, this may not always be the case, so the following rules apply in that case:

  • You must make your final submission within five days of the due date. Submissions which are more than five days late will not be graded.
  • If your code is submitted late, a 5% per day penalty will apply (fractions of days are rounded up), after five days you will automatically receive a zero.
  • Resubmissions are allowed but late penalties will apply based on the last submission.
  • The lowest of your Phase IV and Phase IV Makeup grades will be dropped.
